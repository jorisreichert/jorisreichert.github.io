The tutorial

WordCount

Following the steps of the tutorial was quite easy. The only issue I encountered was when I tried to run the WordCount code, as I already had an output directory. As I was already in the folder where the input and output folders were located (in /opt/hadoop) I removed this using the following command: hadoop fs -rm -r output/. After this I could attempt to run the WordCount code again by doing: hadoop jar wc.jar WordCount input output

Once hadoop is finished, we have to do a few more steps to analyse the output. First, we want to write the output from the HDFS filesystem to the docker container. All output is written in the output folder. In it we find two files: One empty file titled _SUCCESS that indicates the program has succeeded, and one 'file' that contains the output titled part-r-00000 To retrieve this file from the HDFS we do: hdfs dfs -get output/part-r-00000 output-file Now we have the output written in the normal filesystem of our docker container in the file 'output-file'.

If we want to move this file from our container to our host system, we open a terminal in the location where we want to write the file and do: sudo docker cp hello-hadoop:/opt/hadoop/output-file output-file

After having moved the output out of the hdfs, we can clear up the output from the hdfs to make room for other programs using: hadoop fs -rm -r output

Blog post questions

Hadoop commands

What happens when you run the Hadoop commands (hdfs dfs etc.) in the first part of the tutorial?

There are several hdfs commands we used in the first part of the tutorial.

hdfs dfs

The hdfs dfs command can be used to interact with the hadoop filesystem. It runs a filesystem command with which you can put files from your environment in the hdfs and vice versa. To put files from our environment in hdfs we have used: hdfs dfs -put 100.txt input The -put flag indicates that we want to put the first file (here: 100.txt) into the hdfs, giving it the name of the second file (here: input). We have also done it the other way around to retrieve output from the hdfs to our environment: hdfs dfs -get output/part-r-00000 output-file1 The -get flag indicates that we want to get the first file (here: output/part-r-00000) from the hdfs to our environment and write the output in the second file (here: output-file1).

Romeo and Juliet

Does Romeo or Juliet appear more often in the plays? ++ Can you answer this question making only one pass over the corpus? Upon inspection of the corpus of 100.txt (the entire works of William Shakespeare) we see that names can be written in a variety of ways depending on the use. In a play an appearance can be seen as either:

  • Entering the stage, eg. [Enter Juliet.]
  • Speaking, indicated by a name in capital letters followed by a period, eg. ROMEO.
The first option can be harder to count as we can't be sure that every time a character enters the stage there will be a closing bracket ']' as it is possible that multiple people enter the stage simultaneously, eg. [Enter Lady Capulet, and Nurse.] To prevent having to look at a bigger context we will focus on the second option. To count the occurrences of Juliet and Romeo speaking we make a slight adaptation to the WordCount script as provided in the assignment. The mapper will now only write a word when the word is either 'ROMEO.' or 'JULIET.'. if(word.toString().compareTo("ROMEO.")==0 || word.toString().compareTo("JULIET.")==0){ context.write(word, one); }

This approach requires us to pass over the corpus only once after taking the results from the hdfs we see the following occurrences:

  • JULIET. 125
  • ROMEO. 163
